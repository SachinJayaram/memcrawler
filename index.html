The Problem
AI memory isn't one thing.
And it doesn't always go away.
AI platforms accumulate information about you across multiple surfaces — explicit saved facts, chat history references, imported data, local agent files — with different deletion flows, different persistence behaviors, and zero unified audit trail.
💉

Memory Injection Attacks

Researchers have demonstrated MINJA attacks where malicious content can plant persistent instructions in your AI agent's memory through normal interactions — silently shaping future responses.

👻

Deletion Doesn't Mean Gone

Deleting chat history doesn't always delete saved memory. Disabling memory doesn't always clear prior records. What you think you deleted may still be influencing outputs.

🗂️

Seven Persistence Surfaces

Saved memory, chat history reference, exports, imports, local agent files, tool stores, and unknown residuals. Every surface has different behavior. No tool modeled all of them — until now.

🔐

Secrets Leak Into Memory

Agent frameworks routinely store API keys, credentials, and PII in local memory files with no encryption, no TTL, no access controls. Developer mode scans for all of it.

Features
Everything you need to audit AI memory.
Security

MINJA Detection Engine

20+ detection rules targeting Memory INJection Attack patterns — future instructions, wallet redirection, tool manipulation, identity substitution, bridge-step phrasing, suppression directives.

Privacy

Full Memory Inventory

Every detectable record listed with platform, persistence surface, confidence level, and suspicion score. Filter, search, sort. Export as JSON or HTML report.

Security

Deletion Gap Detection

Actively verifies that records you've deleted are actually gone. Compares pre/post deletion snapshots. Surfaces records that persist when they shouldn't.

Privacy

Snapshot Diffing

Compare memory at two points in time. See exactly what was added, removed, or modified — and when. Correlate suspicious additions with specific interactions.

Developer

Memory Policy Linter

Treats agent memory safety as a code quality problem. Flags missing TTLs, stored credentials, overly broad writes, and missing consent flows before you ship.

Privacy

Provenance Confidence

Every record carries a confidence level — Confirmed, Strong Inference, Weak Inference, Unknown. You always know how certain the data is. No false precision.

Developer

Plugin Adapter System

Community can add support for any AI platform or agent framework by implementing a single interface. ChatGPT, Claude, JSON, Markdown, SQLite, custom agents — all supported.

Security

Temporal Anomaly Detection

Detects memory burst patterns — 5+ records in a 60-minute window — that suggest automated injection rather than organic memory accumulation.

Privacy

Fully Offline

All analysis runs locally. No telemetry. No required account. No cloud. Designed to earn trust by requiring none. Your memory data never leaves your device without explicit export.

Honest Limitations
We tell you what we can't see.
Every scan result includes a capability statement. We don't pretend to see server-side data. We don't claim certainty where we have inference. This is not a marketing choice — it's the core of what makes this tool trustworthy.

✓ What Mem Crawler Can See

  • Explicitly saved memory (via official export)
  • User-visible memory UI records
  • Local agent memory files (JSON, MD, SQLite)
  • Imported memory records
  • Memory diffs between two snapshots
  • Deletion gaps (records present after reported deletion)

✗ What Mem Crawler Cannot See

  • Server-side retention after deletion
  • Model weights or training influence
  • Private API internals
  • Memory not included in official exports
  • Encrypted or obfuscated memory stores
  • Platform behavior not reflected in user-accessible data
